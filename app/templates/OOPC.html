Encapsulation is one of the cornerstones of OOP is encapsulation. It refers to the combination of data and procedures that operate on it. Encapsulation is a technique for hiding the values or state of a structured data object within a class, limiting unauthorized access.
Here is an example of encapsulation:
@classmethod
def create(cls, v: tuple):
     """ factory method"""
     return cls(v)
This is considered encapsulation because we are hiding the tuple value from the user. The tuple is pinned down onto the class.

Polymorphism is the method in an object-oriented programming language that performs different things as per the object's class, which calls it. With Polymorphism, a message is sent to multiple class objects, and every object responds appropriately according to the properties of the class.
Here is an example of Polymorphism
# Polymorphism
def add(x, y, z = 0):
     return x + y+z
# Driver code
     print(add(2, 3))
     print(add(2, 3, 4))
The output would be 5 and 9. This is an example of polymorphism because there are two different inputs for the same class when no value passed in it resorts to 0, if you see when you pass in a value for z it gets overridden.

Abstraction is a notion in object-oriented programming that "shows" just the most important features while "hiding" the rest. The basic goal of abstraction is to keep consumers away from superfluous information. It is one of the most crucial OOP concepts.
Here is an example of Abstraction:
for value in self.values:
     result = result * value #Abstraction
return result
It is consider Abstraction because it keeps the user away from the unneeded background information for multiply and only gives what is needed, keeping it easy to follow.
