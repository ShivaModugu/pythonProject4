Separation of Concerns

Separation of concerns is a computer science design principle that divides a program into independent components. Each section deals with a specific problem, a set of data that has an impact on the coding of a computer program. It is important because the lack of duplication and singularity of purpose of the individual components render the overall system easier to maintain. Basically separation of concerns tries to reduce the amount of coupling in the classes and increase the cohesion.
Coupling is the degree of interdependence between software modules.
Cohesion is the degree to which the elements inside a module belong together.
In the example below you will see a good example of cohesion, where the class inheriting values from the superclass. This reduces coupling because the modelue is where the program is being run.
Example:

from calculator.calculations.calculation import Calculation

class Addition(Calculation):
     """ calculation addition class"""
     def get_result(self):
          """get the addition results"""
           sum_of_values = 0.0
           for value in self.values:
               sum_of_values = value + sum_of_values
           return sum_of_values
