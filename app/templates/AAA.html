AAA & Unit Testing

AAA stands for Arrange, Act, and Assert. When you do run these tests you run will start with "arranging" or setting up the tests. Then we are going to "act" and call the code we want to test. Finally, we are going to "assert" that the code we called acted in a way we could expect. So again, We "arrange" or we set up any variables, conditions we might need. We then "act" and execute the code we are interested in testing and finally we "assert" or check that the code is behavior in the way we would expect.
You need to know the errors properly before launching the app or website. This allows a user to commit code without having the fear of it failing down the road.
Example of Testing, type into a python editor and run it for yourself.
def greet(person):
     return "Hi {name}".format(**person)
def test_greet():
     bob = {"name":"Bob"} #Arrange
     greeting = greet(bob) #Act
     Assert greeting == "Hi Bob" #Assert
When you type pytest command into the terminal it should show passed.

After that, we'll need to unit test cases that go beyond the unittest case.
Example of Unit Testing
import unittest
class TestStringMethods(unittest.TestCase):
     def test_isupper(self):
         self.assertTrue('FOO'.isupper())
         self.assertFalse('Foo'.isupper())
